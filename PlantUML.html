Tutorial: Using PlantUML for Class Diagrams

PlantUML is a simple yet powerful tool for creating class diagrams and other UML diagrams. It uses a plain-text language to define the diagram, making it accessible for developers familiar with text-based tools. This tutorial will walk you through the basics of creating class diagrams with PlantUML.

While you can install and use PlantUML locally, I suggest using an online IDE like PlantText (https://www.planttext.com).

Add the following to begin:

xxxxxxxxxx
@startuml
@enduml

All your diagram definitions go between @startuml and @enduml.

Basic Syntax for Class Diagrams

1. Creating a Class

Define a class with its attributes and methods:

xxxxxxxxxx
@startuml


class ClassName {


  + publicAttribute: Type


  - privateAttribute: Type


  # protectedAttribute: Type


   


  + publicMethod(): ReturnType


  - privateMethod(param: Type): ReturnType


}


@enduml

The result:

 

2. Relationships Between Classes

• Inheritance: Use <|-- for inheritance.

• Composition: Use *-- for composition.

• Aggregation: Use o-- for aggregation.

• Association: Use -- for general association.

• Dependency: Use ..> for dependency.

 

Example:

xxxxxxxxxx
@startuml


class Parent


class Child


Parent <|-- Child /'Inheritance'/


Parent *-- Child  /'Composition'/


Parent o-- Child  /'Aggregation'/


Parent -- Child  /'Association'/


Parent ..> Child  /'Dependency'/


@enduml

3. Interfaces

Use the interface keyword for interfaces.

 

xxxxxxxxxx
@startuml


interface InterfaceName {


  + methodName(): ReturnType


}






class ImplementationClass


InterfaceName <|.. ImplementationClass


@enduml

4. Abstract Classes

 

Use abstract to define an abstract class.

 

xxxxxxxxxx
@startuml


abstract class AbstractClass {


  + abstractMethod(): void


}


@enduml

Adding Details

1. Grouping Classes with Packages

 

Packages help organize your diagram:

 

xxxxxxxxxx
@startuml


package "Package Name" {


  class Class1


  class Class2


}


@enduml

 

The result:

 

2. Annotations

 

Add notes or labels to explain elements:

xxxxxxxxxx
@startuml


class ClassName


note left of ClassName : This is a note


@enduml

The result:

 

3. Visibility Modifiers

 

• + for public

• - for private

• # for protected

• ~ for package-private

 

Example: Full Class Diagram

 

xxxxxxxxxx
@startuml


package "University System" {


  class Student {


    + name: String


    + id: int


    + enroll(course: Course): void


  }






  class Course {


    + courseName: String
 
    + courseCode: String


    + addStudent(student: Student): void


  }






  class Professor {


    + name: String


    + teaches(course: Course): void


  }






  Student -- Course : "enrolls in"


  Professor -- Course : "teaches"


}


@enduml

Tips for Effective Diagrams

1. Keep diagrams simple and focused on a specific context.

2. Use meaningful names for classes, methods, and relationships.

3. Group related classes into packages for clarity.

4. Use annotations to provide context where needed.
